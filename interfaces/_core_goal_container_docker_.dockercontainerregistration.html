containers

containers: DockerGoalContainer[]

Overrides GoalContainerSpec.containers

  • Defined in lib/core/goal/container/docker.ts:71

Containers to run for this goal. The goal result is based on the exit status of the first element of the containers array. The other containers are considered "sidecar" containers provided functionality that the main container needs to function. The working directory of the first container is set to ContainerProjectHome, which contains the project upon which the goal should operate.

This extends the base containers property to be able to pass additional dockerOptions to a single container, eg. '--link=mongo:mongo'.

Optional dockerOptions

dockerOptions: string[]
  • Defined in lib/core/goal/container/docker.ts:80

Additional Docker CLI command-line options. Command-line options provided here will be appended to the default set of options used when executing docker run. For example, if your main container must run in its default working directory, you can include "--workdir=" in the dockerOptions array.
